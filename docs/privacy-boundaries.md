# Privacy Boundaries

These tools are meant to share reusable patterns, not private context.

Keep out of public git:

- Credentials, tokens, cookies, and browser/session state
- Mailbox contents, inbox exports, screenshots, HTML dumps, and generated summaries
- Private local paths, account identifiers, channel IDs, and machine-specific values
- Named people or organizations from Francisco’s private context
- Concrete private alert filters, senders, terms, or matching rules
- Runtime state, logs, caches, debug outputs, and dependency folders

Use public-safe substitutes:

- Placeholder values such as `example.com`, `Routine notice`, or `Selected sender`
- Config templates rather than real config files
- Generic setup notes instead of private operational details
- Short explanations of the pattern, not the private reason it exists
